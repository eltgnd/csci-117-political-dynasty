# Philippine Clan Watch Dashboard

An interactive data visualization platform for studying political dynasty networks in Philippine provincial elections. Built with Streamlit, Plotly, NetworkX, and pyvis.

---

## What it does

The dashboard lets you explore how political families consolidate power across Philippine provinces from 2004 to 2025. It models elected officials as a network where edges represent inferred family ties (derived from shared surnames and middle names), then surfaces four composite dynasty indicators — HHI, CGC, CCD, and ACC — to quantify how entrenched dynasties are in any given province and year.

**Pages:**

- **About the Project** — methodology, mathematics behind the indicators, and how to read each chart.
- **Provincial Analysis** — dynasty network graph, clan heatmap, indicator trends, and top politicians by network influence for any province and year.
- **Politician Search** — look up any politician by name, browse their election history, and explore their ego-network of family connections.

```

config.py                       # Shared constants and province mappings
data_loader.py                  # Cached data loading helpers

backend_math.py                 # Core graph and indicator math

pages/
    landing.py                  # About / methodology page
    provincial.py               # Provincial Analysis page
    search.py                   # Politician Search page

components/
    provincial_view.py          # Provincial chart rendering
    search_view.py              # Search and ego-graph rendering

data/
    political_dynasty.csv       # Main dataset (place here)
    precomputed_indicators.csv  # Generated by precompute_indicators.py
    ph_provinces.geojson        # Philippine province/municipality GeoJSON
```

## Running locally

### 1. Prerequisites

- Python 3.10 or higher
- pip

### 2. Clone or download the project

```bash
git clone <your-repo-url>
cd philippine-clan-watch
```

Or simply unzip the project folder and `cd` into it.

### 3. (Recommended) Create a virtual environment

```bash
python -m venv venv

# macOS / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 4. Install dependencies

```bash
pip install -r requirements.txt
```

### 5. Place data files

Put the following files inside the `data/` folder (create it if it does not exist):

| File | Description |
|---|---|
| `political_dynasty.csv` | Main dataset — must be placed at the project root, **not** inside `data/` |
| `data/ph_provinces.geojson` | Philippine province/municipality GeoJSON (municipality-level features) |

The CSV must have these columns: `First Name`, `Middle Name`, `Last Name`, `Position`, `Position Weight`, `Party`, `Year`, `Province`, `Region`, `Community`.

### 6. Precompute dynastic indicators (strongly recommended)

This step runs the computationally heavy indicator math offline so the app loads instantly. Without it, charts will still work but will be noticeably slower.

```bash
python precompute_indicators.py
```

This writes `data/precomputed_indicators.csv`. Re-run whenever the dataset is updated.

For faster processing on a multicore machine:

```bash
python precompute_indicators.py --workers 4
```

### 7. Run the app

```bash
streamlit run app.py
```

The app will open in your browser at `http://localhost:8501`.

## Deploying to Streamlit Community Cloud

1. Push the project to a public (or private) GitHub repository.
2. Go to [share.streamlit.io](https://share.streamlit.io) and sign in.
3. Click **New app** → select your repository, branch, and set the main file path to `app.py`.
4. Add `political_dynasty.csv` and `ph_provinces.geojson` to the repo (or use Streamlit Secrets / external storage for large files).
5. The `requirements.txt` at the project root is picked up automatically.

> **Note on large files:** If the dataset exceeds GitHub's 100 MB limit, consider hosting it on Google Drive or an S3 bucket and fetching it at startup via a custom `load_data()` implementation.
